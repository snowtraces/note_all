# 个人知识收集与检索系统 (Note All)

## 1. 功能背景

手机上的一些灵感、网页、笔记收藏，目前无好的统一归档方案。这些碎片信息往往是以图片或截屏为主，导致后续在归档中提取有用信息非常麻烦。
我们需要一个解决方案，要能更无感、便捷地收集和归档内容，并且具备自动解析图片信息的能力，最终方便多端（Android、Web）进行跨设备的全文检索。

## 2. 功能规划

为满足“多端便捷收集”、“自动化信息提取”、“轻便检索”的目标，系统划分为以下三个核心子系统：

### 2.1 收集端 (Ingestor)
*   **Android 端**
    *   **全局系统分享**：支持将相册图片、浏览器链接、各类APP文本通过安卓系统自带的“分享”面板一键发送至本应用。
    *   **剪贴板嗅探**：App启动时自动嗅探剪贴板内容（文本/图片链接），提示快速保存。
*   **Web/PC 端**
    *   **快速上传通道**：管理后台提供常规的拖拽上传和粘贴上传（CTRL+V）能力。
    *   **PC 右键上传（PC Client）**：Windows 原生客户端，注册到资源管理器右键菜单，右击任意图片文件即可一键上传至服务器，无需打开浏览器。
    *   **PC 截图上传（PC Client）**：按下全局热键 `Alt+Q`，唤起全屏截图遮罩，鼠标拖拽框选任意区域后自动上传；支持多显示器，遮罩覆盖鼠标所在屏幕；上传结果气泡通知（3s 自动消失）。

### 2.2 处理中枢 (Processor) - *核心亮点*
图片和截图上传后，系统在后台进行异步解析：
*   **基础 OCR 识别**：提取截图/图片中的所有可用文本，持久化落库，用于日后关键词检索。
*   **AI 结构化提取 (VLM/大模型辅助)**：
    *   **自动标签提取 (Auto-Tagging)**：识别图片主题（如：代码、菜谱、票据、好物推荐）。
    *   **自动命名与摘要**：生成一句话的图片描述摘要，替代传统的“无意义命名”。

### 2.3 消费与管理 (Consumer)
*   **收件箱机制 (Inbox)**：所有新收集的内容首选进入收件箱，处理并确认无误后归入“特定知识库/标签”。
*   **多维度融合检索**：支持对“OCR识别文本”、“AI生成的摘要”、“标签”、“归档时间”进行综合搜索。
*   **图文对照视图**：Web与Android端查看详情时，提供清晰的对齐视图，方便用户摘抄或复制文本。

## 3. 实现骨架设计

基于 KISS 原则及极致轻量化的多端开发需求，系统完整技术栈选型如下：

### 3.1 客户端 (Client)
*   **Web端**：`React` (提供现代化的组件化视图管理与生态支持，配合 TailwindCSS 快速构建响应式界面)。
*   **Android端**：`Flutter` 构建，便于一套代码高质量运行，并能快速接入 Android 原生的“系统分享面板”与“剪贴板读取”插件能力。
*   **PC端（Windows）**：纯 `Golang` 实现的轻量托盘程序（`pc_client/`）。
    *   注册 Windows 右键上下文菜单，图片文件一键上传。
    *   全局热键 `Alt+Q` 唤起截图工具，框选后自动上传（支持多显示器、DPI 缩放）。
    *   编译产物为单一 `.exe`，无额外运行时依赖。

### 3.2 服务端 (Backend API)
*   **核心开发语言**：`Golang` (高并发、编译单文件、极低内存占用，非常契合私有部署与轻量化项目)。
*   **数据库**：`SQLite` (零配置单文件数据库，不仅减轻运维负担，还可通过 FTS5 插件实现高效的全文检索，完美贴合知识库需求)。
*   **文件存储**：`自研文件服务 (基于 snow_storage.go)`，避免引入 MinIO 等重型三方中间件，保持单体部署纯粹性。

### 3.3 人工智能与提取组件 (AI & OCR)
*   **OCR**：接入 `百度飞桨 (PaddleOCR)`，可考虑直接嵌入或作为独立微服务调用，提供高准度的文本提取能力。
*   **AI 结构化能力**：对接 `文心大模型 (ERNIE)` 等提供的大模型 API，用于将零散 OCR 文本整合为摘要标签，实现自动化结构分析。

## 4. 快速启动 (Quick Start)

### 4.1 启动后端 (Backend)

后端基于 Golang，使用 SQLite 作为本地数据库，只需确保拥有 Go 环境即可一键启动：

```bash
cd backend

# 第一步：复制配置文件并按需填入千帆大模型 ApiKey
cp config.json.example config.json

# 第二步：安装依赖并运行
go mod tidy
go run main.go
```

> 后端服务默认在 `http://localhost:8080` 启动，图片与数据库均存放在 `backend/` 目录下。

### 4.2 启动前端 (Frontend)

前端由 React + Vite 构建，需要 Node.js 环境（推荐 Node 18+）：

```bash
cd frontend

# 第一步：安装所有依赖
npm install

# 第二步：启动开发服务器
vite run
```

> 启动成功后，在浏览器中打开 `http://localhost:3000`，即可体验图片上传与 AI 自动解析检索功能。

### 4.3 启动 PC 客户端 (PC Client)

PC 客户端基于 Golang，编译需要 Go 1.21+ 与 GCC（MinGW-w64，用于 CGo）：

```powershell
cd pc_client

# 第一步：复制配置文件并填写后端服务地址
cp config.json.example config.json

# 第二步：一键编译
.\build.ps1
```

> 首次运行 `dist\note_all_pc.exe` 后，在系统托盘图标菜单中点击 **「注册右键菜单」** 完成初始化。
> 此后在资源管理器中**右击任意图片文件**，选择 **「上传到 Note All」** 即可一键上传。
>
> **截图上传（推荐）**：程序驻留托盘后，随时按 **`Alt+Q`** 即可：
> 1. 屏幕出现半透明遮罩（自动定位到鼠标所在显示器）
> 2. 鼠标拖拽框选目标区域
> 3. 松开左键 → 自动截图并上传
> 4. 右上角弹出上传结果通知（3 秒后自动消失）
> 5. 按 `ESC` 可随时取消
>
> 也可通过命令行直接调用（适合脚本 / 快捷键触发）：
> ```powershell
> .\note_all_pc.exe --upload "C:\path\to\image.png"
> ```
