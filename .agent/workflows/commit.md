// turbo-all

1. **清理文件**:
   - 使用文件系统工具查找新生成的调试脚本信息与冗余文件。
   - **过滤**: 排除已暂存 (staged) 的文件；排查处于 `.gitignore` 中被忽略的文件。
   - **确认**: 列出这些拟删除文件，并**必须在这里停止并询问用户确认**（不要自动删除）。
   - 在用户明确回复确认后，再执行删除操作。

2. **文档维护**: 
   - 使用文件读写工具创建或更新 `./changelog.md` 文件。
   - 根据本次的代码变更（如果涉及）更新 `docs/` 目录下的相关文档。

3. **Git 提交准备**:
   - 运行命令 `git status -s` 来检查暂存区状态。
   - 如果发现 **已有 staged（带有 M/A/D 等暂存标记标识）的文件**：**跳过** `git add .`。
   - 否则（当暂存区为空时）：运行命令 `git add .` 将所有变更暂存。

4. **生成 Commit 信息**:
   - 根据工作上下文拼装提交信息格式: `<type>(<scope>): <subject>`。
   - **重要规则**: Subject **强制使用中文**。
   - 示例提交信息: `fix(auth): 修复登录超时问题`（或 `feat(Share): 支持多图分享`）。

5. **提交与推送**:
   - 运行命令 `git commit -m "<Commit信息>"` 提交代码。
   - 运行命令 `git pull --rebase` 同步线上代码。
   - 运行命令 `git push` 将代码推送到远程仓库。
