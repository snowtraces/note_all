# Note All 研发路线图 (Roadmap)

基于竞品分析与核心理念"无感收集、AI 即刻懂"，制定以下新功能规划及研发路线图。

## ✅ 已完成阶段

### Phase 1: 浏览器剪藏扩展 (Browser Extension)
* **状态**: ✅ 已完成
* **成果**: 补全了 PC 工作流中的网页信息抓取拼图。支持划词右键剪藏及 Popup 快捷输入。

### Phase 2: VLM (视觉大模型) 图像深度语义感知
* **状态**: ✅ 已完成
* **成果**: 接入多模态大模型，实现了图片内容的深度描述与 OCR 文本的智能融合。

### Phase 3: 智能记忆拼图 (Serendipity Review)
* **状态**: ✅ 已完成
* **成果**: 实现了"灵感碰撞"功能，AI 自动串联随机碎片生成启发性随笔，盘活了死知识。

### Phase 4: 自动隐式双链 (Auto-Linking)
* **状态**: ✅ 已完成
* **成果**: 建立了基于标签交集的关联发现机制，碎片详情页可自动提示相关内容，实现隐式双链。

---

## 🚀 新功能规划（基于竞品分析）

> 竞品参考：Readwise Reader、Mem.ai、Reflect Notes、Obsidian、Capacities

### Phase 5: 语音快速录入 (Voice Memo Ingestion)
**核心洞察**：碎片化场景中，开车/走路时最高效的输入是语音。竞品 Mem.ai 的 Voice Mode 是核心卖点。

* **功能设计**:
  - Android 端新增「语音录入」按钮（悬浮快捷键），录音后自动调用后端 ASR（语音转文字）
  - 后端集成云端 ASR 服务（百度/阿里/OpenAI Whisper）
  - 转录文本走现有 LLM 流水线自动生成摘要与标签
  - PC 端支持 `Alt+V` 热键触发浮窗录音

* **价值**: 打通"手机录音"这一最高频无感录入场景。
* **复杂度**: ⭐⭐⭐（中）

---

### Phase 6: RSS / 邮件订阅源收集 (Feed Subscription)
**核心洞察**：Readwise Reader 的第二大功能是 RSS + Newsletter 订阅，让内容主动送上门。

* **功能设计**:
  - 后端实现 RSS Feed 解析器，支持添加订阅源 URL
  - 定时爬取（每小时）并自动入库新文章，走现有 URL 摘要流水线
  - Web 端新增「订阅管理」页面（增删改查 Feed），以及「未读」视图
  - 支持邮件转发收集（生成唯一收件地址，邮件发到该地址自动入库 Newsletter）

* **价值**: 从"主动采集"升级为"被动接收 + 主动采集"双模式，构建个人信息流。
* **复杂度**: ⭐⭐⭐⭐⭐（高 · 推荐拆分子任务）

---

### Phase 7: 知识图谱视图 (Knowledge Graph View)
**核心洞察**：Obsidian 的 Graph View 是其最具辨识度的杀手锏功能，直观展现知识网络。

* **功能设计**:
  - Web 端新增「图谱」视图（使用 `d3.js` 或 `vis.js` 绘制）
  - 节点 = 笔记条目，边 = 共享标签（基于现有 `note_tags` 表，零额外成本）
  - 支持节点点击跳转详情、标签过滤缩放
  - 图谱作为与 List 视图并列的第三种查阅维度

* **价值**: 从线性列表升级为网状知识地图，直观洞见知识密度与盲区。
* **复杂度**: ⭐⭐⭐（中，数据已具备，主要是前端渲染）

---

### Phase 8: 自定义 AI Prompt 模板 (Custom AI Prompt)
**核心洞察**：Reflect Notes 和 Readwise Ghostreader 均开放 Prompt 自定义，极大增强工具适配性。

* **功能设计**:
  - 设置页新增「AI 处理模板」管理，内置几个预设（学习笔记、代码片段、阅读书摘）
  - 用户可自定义摘要提示词、标签规则、格式要求
  - 手动触发对单条笔记执行「重新 AI 处理」（使用当前激活模板）
  - 模板配置持久化到本地 SQLite

* **价值**: 让 AI 处理从"通用"变为"为我定制"，适配不同知识工作者的场景。
* **复杂度**: ⭐⭐（低，主要是 UI + 配置持久化）

---

## 📋 优先级建议

| 阶段 | 功能 | 优先级 | 理由 |
|------|------|-------|------|
| Phase 7 | 知识图谱视图 | 🔴 高 | 数据已具备，实现成本低，视觉冲击力强 |
| Phase 8 | 自定义 AI Prompt | 🟡 中 | 实现简单，能显著提升高级用户满意度 |
| Phase 5 | 语音快速录入 | 🟡 中 | 补全 Android 端最后一个高频场景 |
| Phase 6 | RSS 订阅源 | 🟢 低 | 实现复杂度最高，可作为长期目标 |

---

*注：遵循渐进式开发原则，优先交付高性价比 (低成本高价值) 功能。*
