# Note All 研发路线图 (Roadmap)

基于竞品交叉对比分析，重新规划研发路线，补齐核心差距，强化竞争优势。

---

## 设计原则

1. **渐进式交付**：高性价比功能优先，逐步补全核心差距
2. **依赖有序**：基础能力 → 进阶能力 → 高级体验
3. **保持主线**：延续"无感收集、AI 即刻懂"核心理念
4. **阶段验收**：每个阶段有明确交付目标

## 🚀 Phase A: AI 能力深化

> **目标**: 强化"AI 即刻懂"核心竞争力，让 AI 从被动响应升级为主动思维伴侣

---

### Phase A1: 视频深度解析 ⏸️ [搁置]

**核心洞察**: YouTube、播客是现代碎片知识的主战场，音视频信息密度大但难以检索。

**优先级**: 🔴 P0 | **复杂度**: ⭐⭐⭐ 中 | **状态**: ⏸️ 搁置

**搁置原因**: 当前模型转录/理解能力不足以支撑高质量输出，待技术成熟后重启。

**功能设计**:

- **URL 智能转录**: 提交 YouTube/Bilibili 链接，自动抓取文稿并生成分段摘要
- **语义节点定位**: 点击摘要观点，直接跳转视频对应秒数
- **关键帧 OCR**: 自动采样视频 PPT/黑板画面，识别文字并入库
- **内置播放器**: 支持回溯播放，片段标记

**验收标准**: 提交视频链接 → 生成可搜索文稿 → 点击观点跳转秒数

**技术依赖**: 后端异步任务队列、Whisper/API 转录服务、媒体处理模块

---

### Phase A2: AI 主动启发

**核心洞察**: Mem.ai 证明 AI 可以主动启发而非被动搜索，浏览时推送关联提示能激发思考。

**优先级**: 🔴 P0 | **复杂度**: ⭐⭐⭐ 中

**功能设计**:

- **启发式推送**: 浏览碎片时，AI 推送"这与 X 知识有何联系？"
- **相关推荐面板**: 详情页自动展示相关碎片（基于向量相似度）
- **定期回顾提醒**: 每日/每周 AI 生成知识回顾摘要
- **矛盾检测**: 跨碎片发现观点冲突或逻辑缝隙

**验收标准**: 详情页显示 AI 主动推送的关联提示 + 相关碎片推荐

**技术依赖**: 向量检索优化、推荐算法、定时任务调度

---

## 📊 Phase B: 知识关联增强

> **目标**: 补全基础能力差距，让碎片组织更灵活、关联更主动

---

### Phase B1: 显式双向链接

**核心洞察**: Obsidian/Logseq 的核心竞争力，用户主动创建链接比隐式发现更可控。

**优先级**: 🟡 P1 | **复杂度**: ⭐⭐⭐ 中

**功能设计**:

- **Wiki 链接语法**: `[[碎片标题]]` 自动创建链接
- **反向链接面板**: 详情页显示"哪些碎片引用了我"
- **链接预览**: Hover 显示目标碎片摘要卡片
- **图谱路径追踪**: 高亮显示双链路径

**验收标准**: 输入 `[[某标题]]` → 自动关联 → 目标页显示反向链接

**技术依赖**: Markdown 解析扩展、数据库链接关系表

---

### Phase B2: 数据库多维视图

**核心洞察**: Notion 杀手锏，海量碎片需多维度组织，长列表体验随数据增长变差。

**优先级**: 🟡 P1 | **复杂度**: ⭐⭐⭐⭐ 高

**功能设计**:

- **看板视图**: 按标签/状态拖拽整理碎片
- **日历视图**: 时间轴回顾，关联日记模式
- **智能文件夹**: 保存复杂筛选条件（如"含 AI 标签且有图片的近 3 天碎片"）
- **属性列编辑**: 自定义元数据字段（来源、优先级等）

**验收标准**: 切换看板 → 拖拽碎片改变状态 → 视图持久化

**技术依赖**: 前端视图组件重构、状态管理优化、筛选引擎

---

### Phase B3: 版本历史系统

**核心洞察**: 编辑安全感是基础需求，支持回滚防止误操作损失。

**优先级**: 🟡 P1 | **复杂度**: ⭐⭐ 低

**功能设计**:

- **版本快照**: 每次编辑自动保存版本
- **Diff 对比**: 版本差异可视化
- **回滚能力**: 选择历史版本恢复
- **编辑时间线**: 显示编辑历史记录

**验收标准**: 详情页显示历史版本列表 → 选择回滚

**技术依赖**: 数据库版本表、存储策略（定期清理旧版本）

---

## 🔌 Phase C: 开放生态

> **目标**: 成为 AI 生态基础设施，打破 App 孤岛

---

### Phase C1: MCP 协议支持

**核心洞察**: Anthropic MCP 协议让个人知识库成为所有 AI 客户端的"外挂大脑"。

**优先级**: 🟡 P1 | **复杂度**: ⭐⭐⭐ 中

**功能设计**:

- **MCP Server**: 实现 Model Context Protocol，供 Claude Desktop 调用
- **语义检索 API**: 提供向量搜索接口，支持外部工具集成
- **自动化联动**: Raycast、n8n 等工具可检索知识库
- **权限与安全**: API Key 管理、访问控制

**验收标准**: Claude Desktop 可调用 Note All 进行知识检索

**技术依赖**: MCP SDK 集成、RESTful API 设计、安全策略

---

## 📰 Phase D: 信息流基建

> **目标**: 从"主动采集"升级为"智能过滤"，构建个性化信息流

---

### Phase D1: Newsletter 聚合

**核心洞察**: Newsletter 是高质量内容源，但散落各处难以管理。

**优先级**: 🔵 P2 | **复杂度**: ⭐⭐⭐ 中

**功能设计**:

- **专属邮箱**: 每用户生成接收地址
- **自动解析**: Newsletter 解析为干净碎片，去除广告/样式
- **AI 摘要**: 提取核心观点，自动打标签
- **去重归档**: 相同来源内容智能合并

**验收标准**: 发送 Newsletter 到邮箱 → 自动入库并生成摘要

**技术依赖**: 邮件服务集成、HTML 解析器、去重算法

---

### Phase D2: RSS 智能订阅

**核心洞察**: RSS 仍是重要信息源，AI 分级可解决信息过载。

**优先级**: 🔵 P2 | **复杂度**: ⭐⭐⭐ 中

**功能设计**:

- **RSS 源管理**: 添加/删除/编辑订阅源
- **自动抓取**: 定时拉取新内容入库
- **AI 智能分级**: 精读/扫读/忽略三级评分
- **原子化拆解**: 长文自动拆解为多卡片

**验收标准**: 添加 RSS 源 → 自动抓取 → AI 分级标注

**技术依赖**: RSS 解析模块、调度任务、分级 Prompt

---

### Phase D3: 任务视图 🔻 [降级]

**核心洞察**: 碎片中常含隐含任务，但独立任务系统 ROI 偏低。作为多维视图的一种，与碎片组织整合更合理。

**优先级**: 🔵 P2 | **复杂度**: ⭐⭐⭐ 中 | **状态**: 🔻 降级（原 A3）

**降级原因**: 实用性中等偏低（非高频刚需）、与核心定位有偏差、外部工具（TickTick）更成熟。延后至 Phase D，与信息流整合。

**功能设计**:

- **轻量任务标记**: 支持 `#todo` 标签 + 状态字段，复用现有筛选机制
- **任务视图**: 作为 B2 多维视图的一种，按状态/优先级组织
- **手动触发识别**: 用户点击按钮，AI 识别碎片中的任务项
- **五态管理**: 待处理 / 进行中 / 已完成 / 已放弃 / 延期
- **碎片关联**: 任务与源碎片双向链接，支持溯源跳转

**验收标准**: 碎片详情页 → 点击提取 → 任务列表 → 状态流转

**技术依赖**: LLM 任务识别 Prompt、多维视图扩展

**备注**: 不做外部联动（TickTick/Todoist），不做独立任务系统，与 B2 多维视图整合

---

## 🎨 Phase E: 高级体验

> **目标**: 增值功能，提升产品差异化与品牌感

---

### Phase E1: 白板交互编辑

**核心洞察**: Heptabase 证明视觉化思维能解决碎片的空间感问题。

**优先级**: 🔵 P2 | **复杂度**: ⭐⭐⭐⭐⭐ 极高

**功能设计**:

- **无限画布**: 拖拽布局碎片卡片
- **手动连线**: 创建视觉关联关系
- **分组 Section**: 区域划分与命名
- **节点聚合/拆解**: 多碎片合并为主题卡片，或拆解原子节点
- **双链同步**: 白板连线与隐式双链自动关联

**验收标准**: 进入白板 → 拖拽碎片 → 手动连线 → 保存布局

**技术依赖**: 前端画布引擎重构（tldraw/fabric.js）、复杂状态管理

**备注**: 复杂度极高，需在 B1/B2 完成后再启动

---

### Phase E2: 音频播客化

**核心洞察**: NotebookLM 证明对话式音频能有效盘活知识，降低复习门槛。

**优先级**: ⚪ P3 | **复杂度**: ⭐⭐⭐⭐ 高

**功能设计**:

- **播客生成**: 选定碎片，AI 生成两人对谈式播客音频
- **每周汇总**: 自动生成周知识回顾语音
- **分享下载**: 播客可下载或分享

**验收标准**: 选择碎片 → 生成播客 → 播放/下载

**技术依赖**: TTS 服务集成（ ElevenLabs/OpenAI ）、音频生成流程

**备注**: 炫技型功能，品牌价值大于实用，时机成熟后启动

---

### Phase E3: 协作与分享体系

**核心洞察**: 团队知识管理需求存在，但需产品定位决策。

**优先级**: ⚪ P3 | **复杂度**: ⭐⭐⭐⭐ 高

**功能设计**:

- **多用户系统**: 用户注册、邀请机制
- **权限管理**: 只读/编辑/管理员分级
- **共享空间**: 知识库/文件夹级别分享
- **协作编辑**: 多人同时编辑（可选）

**验收标准**: 邀请成员 → 共享知识库 → 协作编辑

**技术依赖**: 用户系统重构、权限架构、实时协作引擎

**备注**: 需明确产品定位（个人工具 vs 团队工具）后再决策
