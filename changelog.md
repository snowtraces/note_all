# Changelog

## [Unreleased]
- **JWT 身份验证重构 (系统全链路)**:
  - **后端加密安全层**: 废弃简单的明文密码 Header 校验，引入 `github.com/golang-jwt/jwt/v5` 实现标准的 JWT 鉴权。
  - **登录发放机制**: 新增 `/api/auth/login` 接口，验证密码后签发受保护的 JWT 令牌（有效期 30 天）。
  - **鉴权中间件升级**: `AuthRequired` 现全面支持 `Bearer <JWT>` 解析与签名一致性验证。
- **浏览器插件 (鉴权适配与配置简化)**:
  - **JWT 握手支持**: 更新设置界面逻辑，保存配置时自动向后端换领 JWT 令牌。
  - **配置零负担**: 简化 API 配置项为单一的 `Server URL`，自动补全业务路径。
  - **上下文菜单增强**: 确保右键剪藏文本及图片时均能稳定注入 `Authorization` 凭证。
- **Android (JWT 全模型适配)**:
  - **登录逻辑集成**: `NoteRepository` 新增登录端点支持。
  - **双重存储架构**: `ConfigManager` 支持同时管理明文密码与 JWT 令牌，保障了在令牌过期后的自动静默登录与 UI 回显。
  - **UI 与配置刷新**: 设置弹门具备了即时的登录有效性校验功能。
- **PC 客户端 (鉴权重构与 UI 增强)**:
  - **交互式设置面板**: 新增基于 WinForms 的服务器配置界面，支持即时修改地址与密码。
  - **JWT 自动握手**: 托盘菜单新增“设置”项，点击保存时自动换领后端颁发的 JWT 身份令牌并持久化。
  - **网络层深度适配**: `UploadFile` 与 `UploadText` 全面支持 `Bearer` 令牌注入，确保截图与快捷录入功能的安全性。
  - **按需生成分享链路**: 为笔记碎片引入了基于随机加密 ID 的公开发布机制，支持后台管理与随时撤销。
  - **沉浸式公开展示页**: 
    - 实现深度优化的 **全屏阅读体验 (`PublicSharePage`)**，具备极致的视觉排版。
    - 强化原始素材展示，采用更利于研读的深色容器与高行距排版。
    - 优化了宽屏适配 (`max-w-5xl`) 与文字清晰度 (`opacity: 0.92`)。
  - **分享管理面板**: 在详情页集成 `ShareModal`，支持一键创建、复制链接及管理。
  - **安全性与性能**:
    - 分享接口完全脱离 `AuthRequired` 鉴权，确保访客平滑访问。
    - 在分享路径下自动禁用前端 `/search` 轮询探针，节省资源。

### Added
- **Security (认证系统实装 - Phase 1)**:
  - **单人模式鉴权**: 引入了基于 `sys_password` 的轻量化认证体系，适配单人文档管理场景。
  - **后端安全中间件**: 新增 `AuthRequired` 中间件，强制校验 `Authorization: Bearer` 令牌，有效覆盖所有业务接口。
  - **路由隔离**: 实现了公开接口（登录）与保护接口的逻辑分离，防止认证过程被自身拦截。
  - **前端认证闭环**: 
    - 新增 **高级感登录 UI (`LoginOverlay`)**，采用毛玻璃视效与极简交互。
    - 封装 `client.js` 请求包装器，实现 Token 自动注入与 401 自动登出重载逻辑。
    - 在导航栏 (`NavRail`) 集成手动登出功能，支持完整的会话生命周期管理。

  - **检索与图谱全链路打通**: 确保 `done` 状态的笔记在全局搜索与图谱中依然保持活跃可见。
- **Frontend (交互工作流深度重构)**:
  - **沉浸式批注系统**: 在详情页右侧边栏底部新增 **固定吸底** 的批注区域。

    - 为侧边栏引入分层滚动机制：上部元数据区自由滚动，底部批注区稳固吸附，极大优化了长内容的阅读体验。

- **Android Client (API 对齐)**:
  - **同步状态更新链路**: 在 `NoteApi`、`NoteRepository` 及模型层补全了 `updateNoteStatus` 相关定义，为移动端后续接入批注与状态管理功能完成了基建铺垫。
- **Android Client (稳定性与体验增强)**:
  - **接口超时放宽**: 配置专用 `OkHttpClient`，将读取超时延长至 120 秒，完美适配长耗时的 RAG (AI) 对话流程。

  - **归档决策后置**: 将“自动归档原始素材”复选框移至最终确认保存按钮旁，支持在保存阶段进行最后决策。
  - **即时属性预览 (Portal Hover)**:
    - 为素材卡片引入“冒泡式”预览气泡，通过 React 状态外置管理，利用 `OffsetTop` 动态计算位置，彻底解决了滚动容器导致的 `z-index` 遮挡与边界裁剪顽疾。
    - 预览采用 `PRE` 格式化排版，完美还原 OCR 原始换行与格式；支持鼠标平滑移入气泡进行选词或检阅。
- **Backend (知识维护与搜索增强)**:
  - **正式引入“归档”状态机**: 为 `NoteItem` 模型增加 `is_archived` 字段，并在 `NoteApi` 中实现了逻辑闭环。
    - **默认视图纯净化**: 笔记列表、搜索（非 FTS 常规搜索）及 AI 问答默认过滤已归档内容，保持工作区专注。
    - **搜索触达优化**: 仅在 ID 精确检索时允许触达归档内容，确保知识不丢失。
    - **批量管理支持**: 新增 `BatchArchive` 接口，支持用户快速整理知识库。
  - **搜索意图理解预处理**: 新增 `normalizeQueryForFTS` 处理器。在执行 FTS 搜索前自动剥离“如何”、“怎么”、“呢”、“吗”等自然语言干扰项，极大提升了中文环境下检索词的命中率，解决了问句搜索搜不到内容的痛点。
- **知识实验室 (Knowledge Lab) 持久化重构**:
  - **合成流程“预览-保存”解耦**: 将 `Synthesize` 接口降级为预览模式，仅返回 AI 生成结果；配合前端新增 `SaveSynthesized` 持久化接口，实现先确认、后落库的受控流程。
  - **溯源数据同步**: 在保存合成笔记时，同步建立与原始素材的父子关联，确保 Lineage Tracking 闭环。
- **Generic**:
  - `config.json` 新增 `vlm_model_id` 配置项，支持多模态模型的灵活切换。
- **Backend (RAG 增强与同义词检索)**:
  - **本地同义词扩展**: 集成《哈工大同义词词林扩展版》（约 9 万条数据），支持通过本地 FTS5 引擎在毫秒级内完成查询词的同义词扩展，显著增强了语义覆盖度。
  - **自研分词重写逻辑**: 在执行检索前自动对 query 进行分词并剥离“我、的、是”等停用词，将 query 转化为核心实体 Token 列表。
  - **混合检索权重调优**: 
    - **向量门槛回归**: 将向量相似度门槛回调至 0.3，通过“宽进严重排”确保基础语义召回。
    - **标签检索强化**: 实现基于扩展 Token 列表的 `IN` 匹配，并赋予标签命中 **5 倍** 评分权重。
    - **归档穿透**: 取消了 HybridSearch 对归档状态的硬过滤，确保只要存在显式命中（关键词或标签），归档知识亦能被检索并参与 RAG。
  - **FTS5 鲁棒性优化**: 修复了查询词包含特殊引导符（如 `#`）时触发的 SQLite FTS5 语法错误。
- **Docs**:
  - **RAG 设计文档**: 在 `README.md` 中补充了完整的 RAG 检索设计章节，详细解构了意图探测、混合检索通道及分值加权机制。

- **Android Client**: 清理了不再使用的麦克风图标、圆角形状及相关的 Toast 提示逻辑，保持代码库整洁。
- **Backend**: 经核查后端已确认清理 ASR 相关逻辑，系统回归轻量化设计。

### Added
- **ROADMAP (研发路线图)**: 完成新一轮竞品分析（参考 Readwise, Mem.ai, Heptabase 等），规划并补充了语音录入、RSS 订阅、知识图谱、自定义 Prompt 以及新增的 **视觉思考白板 (Canvas)**、**AI 主动反射 (Reflection)** 与 **多维动态视图 (Views)** 等 Phase 5-11 核心功能路径。
- **知识实验室 (Knowledge Lab - Phase 9-10)**：正式上线面向高阶用户的知识炼金系统，实现从碎片到体系化长文的飞跃。
  - **知识合成引擎 (Synthesize Engine)**：引入多维语义分析算法。支持跨搜索上下文自由选取素材，由 AI 进行深度因果推断与逻辑重组，生成高质量的结构化 Markdown 知识笔记。
  - **三栏式沉浸实验室 (LabView)**：采用深色半透明极客设计。左侧素材篮 (Basket) 持久化缓冲选中项，右侧实时预览合成成果，支持“合成后自动归档”流水线作业。
  - **知识谱系追随 (Lineage Tracking)**：建立了基于多对多关联的溯源体系。合成笔记自动记录所有源碎片 ID，详情页右侧底部动态展示“溯源谱系”，支持一键跳回原始素材。
  - **平滑归档模式 (Archiving)**：支持将处理完的碎片批量移入归档状态。默认主列表仅保留活跃知识点，通过归档机制保持日常工作区的高度整洁，同时保留全局搜索的检索价值。
- **Browser Extension (浏览器剪藏扩展)**: 新增 Chrome/Edge 浏览器扩展，补全 PC 工作流中网页信息抓取的最后一环。支持以下三种采集模式：
  - `Popup 弹窗快捷输入`：点击扩展图标即可快速录入文本或粘贴当前页面链接。
  - `划词右键剪藏`：选中网页文本后通过右键菜单一键剪藏至 Note All，保留原始 URL 溯源。
  - `图片一键采集`：支持右键点击图片直接发送至 Note All，采用浏览器前端下载并模拟上传模式，完美规避反爬虫拦截。
    - **核心渲染优化**：
      - 修复了划词剪藏时推文等文本段落换行完全丢失的顽疾，通过标记预处理策略成功绕过 Turndown 的空白符归一化限制，实现 `pre-wrap` 文本原样换行。
      - **排版布局增强**：优化了对 Flex 布局及内联块级元素的识别。在转换前自动检测 DOM 节点的计算样式，将视觉上处于同一行或具有内联属性的块级标签（如 `div`、`p`）动态降级为 `span`，彻底解决了 Twitter 用户信息等单行多项数据提取后被强制分割换行的痛点；同时在水平排列元素间智能补充空格，确保转换后的 Markdown 内容既不拥挤也不断行。
      - **完善代码块转换**：强制将所有 `<pre>` 标签转换为 Fenced 代码块，并解决了代码块内换行丢失及由于被误转义（如 `\#`, `\_`）导致的源码失真问题；通过全局正则还原逻辑确保代码和正文内容的原样展现。
      - 修复了行首特殊字符（如 `-` 列表符）被误转义为 `\-` 的不一致问题，确保摘录内容视觉排版保持原始状态。
  - 引入 `turndown.js` + `turndown-plugin-gfm.js` 实现页面内容 HTML→Markdown 净化转换。
- **VLM 视觉大模型深度感知 (Phase 2)**: 接入多模态大模型，对上传图片进行深度语义理解。
  - 自动生成图片描述（Description）与内容摘要（Summary），超越纯 OCR 识别能力。
  - VLM 产出与 PaddleOCR 提取文本融合，统一写入结构化知识库。
  - 引入激进压缩策略（分辨率缩至 512px，JPEG 质量 60），最小化 Token 消耗与 API 成本。
  - LLM Prompt 规范化，强制输出严格 JSON 格式 `{description, summary, tags}`。
- **智能记忆拼图·灵感碰撞 (Phase 3 - Serendipity Review)**: 实现"灵感碰撞"功能，盘活长期沉默的碎片知识。
  - 随机抽取多条笔记碎片，由 AI 自动串联生成启发性随笔 / 联想片段。
  - 前端 `EmptyState` 组件新增"灵感碰撞"交互入口，支持"深思"模式切换。
  - 布局固化处理，输入区高度稳定，不因内容区变化而抖动。
- **自动隐式双链 (Phase 4 - Auto-Linking)**: 建立基于标签交集的关联发现机制。
  - 碎片详情页自动检测并展示具有共同标签的相关条目，无需手动维护链接。
  - 后端新增关联查询接口，基于 `note_tags` 标签表计算交集权重并排序返回。
- **自定义 AI Prompt 模板 (Phase 8 - Custom AI Prompt)**: 开放 Prompt 自定义机制，极大增强工具适配性。
  - Web 客户端新增「AI 处理模板管理」设置页（涵盖新建、激活、修改与删除等操作）。
  - 内置了三种高频场景的预设模板：「通用抽取（默认）」、「逻辑排版」与「学习笔记」。
  - 允许随时手动切换模板对指定笔记条目发起「重新 AI 处理」，配合底层 `useDataPoller` 实现无感局部热更新。
  - 实现了新建配置时默认填入「通用抽取」提示词样板，优化了使用便捷性。
- **全景知识图谱视觉与性能重构 (Phase 4 - Knowledge Graph)**:
  - **沉浸式全屏体验**：重构了图谱布局架构，使其完美占满侧边栏右侧全部空间，模拟 Obsidian 的深度思考状态。
  - **物理模拟持久化 (影子图层)**：采用 CSS 显隐控制 (`opacity`/`z-index`) 代替条件渲染，使 D3 物理引擎在后台持续运行，解决了切换视图时图谱重绘和位置丢失的痛点。
